Leetcode 605. 种花问题

发表于 2021-01-01 更新于 2021-12-05

新年新气象 刷题快乐！

题目

假设你有一个很长的花坛，一部分地块种植了花，另一部分却没有。可是，花卉不能种植在相邻的地块上，它们会争夺水源，两者都会死去。
给定一个花坛（表示为一个数组包含0和1，其中0表示没种植花，1表示种植了花），和一个数 n 。能否在不打破种植规则的情况下种入 n 朵花？能则返回True，不能则返回False。

示例 1:
输入: flowerbed = [1,0,0,0,1], n = 1
输出: True

示例 2:
输入: flowerbed = [1,0,0,0,1], n = 2
输出: False

注意:
数组内已种好的花不会违反种植规则。
输入的数组长度范围为 [1, 20000]。
n 是非负整数，且不会超过输入数组的大小。

解题思路

解法一：前中后三指针判断是否可以种花

作为一题难度偏低的贪心算法题目，我们可以借由这题窥视到贪心算法的核心思想，即：每一步都选择当前的最佳选择。

本题中，我们会从左到右依次遍历每个坑（列表的每一个元素），并判断可以种花的坑的数量，最后和传入的n进行大小比较，于是得出结果。

对于每个坑而言，只要满足以下三个条件我们就可以说这个坑是可以种花的：
上一个坑是0(前指针prev)，这个坑是0(当前指针i)， 下一个坑也是0(后指针next)。

边界情况：
当前指针抵达左边界时，指针prev应为True。
当前指针抵达右边界时，指针next应为True。

解法二：单指针

分析

由于题目中提到过“数组内已种好的花不会违反种植规则”，所以解法二中直接使用 i+=2 替代指针prev判定的过程，也因此较解法一节约了近半的比较次数。
