基于Docker创建Nginx负载均衡实例

发表于 2022-04-27 更新于 2022-04-28

最近由于业务需要，被要求测试服务的并发量。在手头上只有一台物理机的情况下，搭建并配置k8s集群是来不及了，命令行直接启动的话操作又不够优雅。于是决定用Docker启动多个容器并以nginx作为服务的总入口，借用其轮询机制测试多容器下的服务并发量。

本文的网络架构如下图所示：
nginx作为集群的流量入口，通过普通轮询将请求依次发送到pod1、pod2、pod3中。假设每个pod响应该请求的时间为1s，那么通过以下网络架构则可以通过水平扩容的方式将并发量提升到3。

启动容器并固定容器IP

既然要在nginx中设置转发，那么容器的IP必须是固定的，否则转发请求也就无从谈起了。

以上命令首先构建了一个名为aiphoto-network的docker网段，其范围为10.10.31.0/24。需要注意的是，构建网段的范围不能与宿主机的ip范围重叠，否则宿主机的DNS解析会失效并且在删除该网段前无法恢复。然后在创建后端服务（aiphoto）时，使用–network参数声明该容器在自定义网段aiphoto-network中，再通过–ip参数指定一个ip地址。 以上命令执行完毕后，我们可以通过docker inspect命令查看容器ip，确为我们手动指定的ip地址。（这里使用-v参数是因为后端服务跟静态资源有强关联，纯粹的无状态应用可以略去）

修改nginx配置文件

众所周知，nginx配置文件有两个：nginx.conf和default.conf。但是从网上获取其模板比较复杂，因此我们这里直接从容器中将其拷贝出来。

启动容器后，通过docker cp命令将容器nginx-test中的/etc/nginx/nginx.conf文件复制到宿主机的/root/nginx_conf/nginx.conf；同理，容器的/etc/nginx/conf.d/default.conf也被复制到宿主机的/root/nginx_conf/nginx.conf。

修改nginx.conf

将upstream修改为10.10.31.11:8013、10.10.31.12:8013、10.10.31.13:8013，分别对应pod-1，pod-2，pod-3。

修改default.conf

修改nginx监听端口为8001和8013，其中8001是主页面，8013是请求转发页面：所有进入8013端口的服务都会转发到appserver，也就是三个后端服务中。

使用配置文件启动nginx，打通网络

启动语句的前半段通过-v参数将本地修改好的配置文件挂载进容器中，并通过-p参数将8013端口转发到宿主机的8013端口上。后半句则是将nginx服务与aiphoto-network网络进行打通，否则身处两个网段的服务是无法相互请求的。

此时从一个nginx实例负载到三个后端服务的网络架构已经实现，发送到localhost:8013的网络请求会通过nginx均衡到三个后端服务中。

查看容器ip

docker inspect 容器ID | grep IPAddress

进入容器查看

docker run -it –entrypoint sh nginx:latest
